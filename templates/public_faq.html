Frequently Asked Questions

What is the Certificate Verification System?
Our Certificate Verification System is a blockchain-based platform that allows educational institutions to generate, store, and verify academic certificates. It ensures the authenticity and integrity of certificates through cryptographic verification and blockchain technology.
How do I verify a certificate?
To verify a certificate, simply upload the PDF certificate file through our verification page. The system will automatically calculate the file's hash and compare it with the stored hash in the blockchain. You'll receive immediate feedback on whether the certificate is authentic.
How are certificates stored securely?
Certificates are stored using multiple layers of security:
  • Files are encrypted before storage
  • Certificate hashes are stored on the blockchain
  • Access is restricted to authorized users
  • All transactions are recorded and immutable
What information is required to generate a certificate?
When generating a certificate, you'll need to provide:
  • Student's full name and ID
  • Degree and major
  • Graduation date
  • Issuer's name and title
  • Institution name
  • Unique certificate ID
Can I modify a certificate after it's been issued?
No, once a certificate is issued and stored on the blockchain, it cannot be modified. This is a security feature to ensure the integrity of the certification process. If a correction is needed, a new certificate must be issued with a new certificate ID.
What happens if verification fails?
If verification fails, it means the certificate's hash doesn't match the stored hash in the blockchain. This could indicate:
  • The certificate has been tampered with
  • The certificate is not in our system
  • The certificate ID is incorrect
  • The file is corrupted
In such cases, please contact the issuing institution for assistance.
